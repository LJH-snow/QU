# 社团招新现场执行清单｜A 方案：先把现场动线理顺

## 项目背景
这是针对“学校中等规模社团下周进行 1 天线下招新”的场景方案。  
已知条件是：现场会比较忙，老成员人手有限，新生对社团不熟。  
因此，这次不追求把所有问题一次性都解决，而是先抓住最容易失控的环节，优先保证招新当天现场不乱、信息不漏、流程能跑通。

---

## 一、我的解题路径

## 1. 目标
- 面向对象：招新当天到场的新生 + 负责接待的老成员
- 核心目标：在 1 天线下招新、老成员有限的情况下，降低现场混乱，提升接待效率，保证新生能顺利完成“了解社团—登记报名—知道下一步”

## 2. 这个场景里最关键的问题

### 1）现场分流不清
- 表现：新生来了以后不知道先看哪里、先问谁、先做什么
- 后果：排队拥堵，等待时间长，很多人中途离开

### 2）老成员职责不清
- 表现：有人一直在讲，有人空着，问题来回转
- 后果：重复解释严重，效率低，现场容易乱

### 3）报名路径不统一
- 表现：有人扫码、有人口头留联系方式、有人记在纸上
- 后果：信息遗漏、重复登记，后续跟进接不住

### 4）讲解顺序不固定
- 表现：不同老成员讲法不一样，重点不一致
- 后果：新生理解成本高，对社团印象模糊

### 5）高峰时段没有预案
- 表现：某一时间段突然来很多人
- 后果：前面全堵住，后面所有环节一起失效

## 3. 简单优先级排序（更紧急 / 更重要）
1. 现场分流不清
2. 老成员职责不清
3. 报名路径不统一
4. 讲解顺序不固定
5. 高峰时段没有预案

---

## 二、为什么选择 A，而不是 B / C

## 为什么选 A：先把现场动线理顺
- A 解决的是招新当天最前端、最容易失控的问题
- 现场动线顺了，讲解内容才能被稳定传达，报名信息才能被完整收集
- 在人手有限的情况下，流程优化比单点优化更有效
- A 最适合做成一份别人当天就能直接照着执行的小交付

## 为什么不先选 B：先把介绍内容讲明白
- B 关注“讲得清不清楚”，但前提是现场先不乱
- 如果新生还在排队、堵在入口、登记路径混乱，再好的内容也传达不完整
- 所以 B 不是不重要，而是当前优先级排在 A 后面

## 为什么不先选 C：先把后续跟进接住
- C 关注“招新后会不会流失”，前提是现场先把人接住、信息先收准
- 如果当天登记都不完整，后续跟进也接不住
- 所以这次会轻量兼顾 C，但不把它作为主突破点

---

## 三、我准备交付什么

- 交付物名称：《社团招新现场执行清单》
- 形式：一页执行清单 + 简化动线流程 + 岗位分工表
- 要求：别人看完就能直接照着执行

## 交付的关键模块
1. 入口接待
2. 快速分流
3. 统一讲解
4. 报名登记
5. 答疑与离场提醒

## 最想控制的 2 个风险点

### 风险点 1：现场拥堵，导致新生流失
- 控制方式：设置分流点；将“随便看看 / 想详细了解 / 直接报名”三类人分开处理

### 风险点 2：登记信息不完整，导致后续跟进失败
- 控制方式：只保留一个统一登记入口；现场确认是否提交成功

---

## 四、这次不做什么

1. 不做复杂的长期留存方案  
2. 不做高度个性化的一对一宣讲设计  
3. 不做复杂的小程序或系统开发  

### 不做的原因
这次重点是解决招新当天的执行问题，而不是做一个完整的招新增长系统。  
先把现场流程跑顺，比同时铺开太多目标更重要。

---

## 五、默认前提假设

1. 当天会有一段明显高峰时段  
2. 至少有 4-5 名老成员可轮值  
3. 可以使用二维码表单、微信群和基础指引物料  

---

## 六、一句话总结
这次的思路是：先守住招新当天的现场秩序，用标准化流程替代一部分人力，再用统一登记把后续衔接住。

# 作品成品｜社团招新现场执行清单

## 1. 使用对象
### 直接使用者
- 招新当天在场的老成员
- 临时负责人
- 协助维持现场的同学

### 服务对象
- 第一次接触社团的新生

---

## 2. 作品目标
让新生在较短时间内完成四件事：
1. 知道这个社团是做什么的  
2. 不需要长时间排队或重复提问  
3. 顺利完成报名登记  
4. 离场前知道下一步安排
